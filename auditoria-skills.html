A meta-lição · escalabilidade

O conteúdo é bom. Falta infraestrutura para mantê-lo verdadeiro.

INVESTIMENTO 1

Fonte única + ponteiros, nunca cópias

  • cada regra vive em um arquivo
  • hoje copy existe em 5 lugares
  • a 6ª cópia é a 1ª a divergir
INVESTIMENTO 2

Gate executável — o "skills lint"

  • links · namespaces · tokens · scripts
  • verificados em CI
  • maior ROI de longo prazo
INVESTIMENTO 3

Precedência explícita entre skills

  • já são 4 vendorizadas de 9
  • sem ordem = ruído ativo
  • README com ordem declarada
Verificado-correto · alta confiança · não mexer

Para não gerar retrabalho

Estas afirmações das skills batem com o código — confirmadas arquivo a arquivo.

  • components.json — base-nova · neutral · lucide · aliases corretos
  • tokens.css — todos os tokens nomeados existem (cat-* · surface* · code* · chart-1..5)
  • index.css — Inter + Roboto + Roboto Mono · importa tokens.css
  • @base-ui/react@1.5.0 presente, sem Radix · cmdk consistente
  • a11y — gate jsx-a11y real · disclaimer honesto · tabela entregue
  • i18n — estrutura index.ts/types.d.ts · 7 JSON 1:1 · pluralização

As skills compõem mal porque o wiring nunca resolve e nada testa a verdade. Resolva o trail, declare a precedência, ligue o lint — e o conteúdo bom para de apodrecer.
